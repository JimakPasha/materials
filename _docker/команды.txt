вещде перед командами пишем sudo или отдельно выполняем настройку, чтобы присать без sudo

проверить функциональность докер
$ sudo systemctl status docker

посомтреть версию
docker -v

посмотреть имэджи
sudo docker images

запустить имедж (с докерХаба в первую очередь)
docker run *******

поиск имеджа (можно просто на докерХабе)
docker search *******

скачать имедж
docker pull *******

проверки назначенного IP-адреса сервера
Сетевая команда ifconfig устарела и заменена командой ip в большинстве дистрибутивов Linux.
ifconfig

посмотреть какие контейнеры сейчас бегут
docker ps

посмотреть какие контейнеры которые бежали
docker ps -a

запускаем докер
(-it интерактивно - мы будем видеть как он бежит)
(-p меняем порт(переадресация), по умолчанию 8080)
(******* название имеджа) 
docker run -it -p 1234:8080 *******

(-d)
docker run -d -p 1234:8080 *******

можно запускать без скачки. не найдёт локально, полезет на докерХаб, скачает и запустить
docker run -d(-it) -p 1234:8080 *******

стирать имеджи
docker rmi ********

но сначала нужно стереть контейнер, посомтреть контейнеры ps -a, взять его номер и стереть
docker rm [длинная айдиха контейнера]
а затем имедж стираем как выше описано

ещё можно остановить докер
docker stop ********

сделать копию контейнера
docker tag [название контейнера копию какого хотим сделать] [название копии]

можно изнутри поменять содержимое
sudo docker exec -t [id контейнера] /bin/bash .................. и тд. (39 мин видоса)
