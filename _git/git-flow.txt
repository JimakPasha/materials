Стратегия бранчевания git flow.
-
master - содержит рабочий код
develop - разрабатываем
feature - обычная таска. Брэнч от дев, мерж в дев. Нэйминг - любой.
release - Брэнч от дев, мерж в дев и мастер. Нэйминг - релиз и номер релиза.(тег с номером релиза)
hotfix - Брэнч от мастреа, мерж в дев и мастер. Нэйминг - хотфикс-
-
+ удобно для распределённых команд
+ для традиционных моделей релизов. Не чаще раза в пару недель
+ удобно для нескольких версий продуктов
-
- не удобно для частых релизов в продакшн
- сложно проверить историю работы (большое число истории коммитов)
- при подготовки релиза много времени может уйти на мердж
